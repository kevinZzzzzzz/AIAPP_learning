## 一、服务器基础环境配置

### 1.1 登录服务器

```bash
# 在本地终端执行，替换为你的服务器 IP
ssh root@你的服务器公网IP

# 示例
ssh root@123.456.789.10
```

> 如果是第一次登录，会提示确认指纹，输入 `yes` 回车即可。
> 密码在云服务商控制台的"重置密码"处设置。

### 1.2 更新系统并安装基础工具

```bash
# 更新软件包列表
apt update

# 升级已安装的软件包
apt upgrade -y

# 安装常用工具
apt install -y curl wget vim git unzip build-essential
```

### 1.3 创建普通用户（安全起见，不建议一直用 root）

```bash
# 创建用户（将 yourname 换成你的用户名）
adduser yourname

# 添加 sudo 权限
usermod -aG sudo yourname

# 切换到普通用户
su - yourname
```

## 三、后端服务部署（FastAPI + Python）

### 3.1 安装 Python 3.10+

```bash
# Ubuntu 22.04 自带 Python 3.10，检查版本
python3 --version

# 如果没有或版本太低，手动安装
sudo apt install -y python3 python3-pip python3-venv
```

### 3.2 安装项目依赖

```bash
# 进入后端目录
cd /home/yourname/projects/deepseek-chat/backend

# 创建虚拟环境
python3 -m venv venv

# 激活虚拟环境
source venv/bin/activate

# 升级 pip
pip install --upgrade pip

# 安装依赖
pip install -r requirements.txt
```

### 3.3 修改配置文件

```bash
# 编辑 config.py，填入真实的 API Key 和允许的前端地址
vim config.py
```

关键修改项：

```python
# config.py 修改示例
DEEPSEEK_API_KEY = "sk-你的真实APIKey"

# CORS 允许的来源 —— 上线后改成你的域名或服务器 IP
ALLOWED_ORIGINS = [
    "http://你的域名.com",          # 如果有域名
    "http://服务器公网IP",           # 直接用 IP 访问
    "https://你的域名.com",          # HTTPS 版本
]
```

### 3.4 测试后端能否启动

```bash
# 确保在虚拟环境中
source venv/bin/activate

# 启动测试
python main.py
```

看到如下输出说明成功：

```
╔══════════════════════════════════════════════╗
║       DeepSeek Chat API                      ║
║  Model:   deepseek-chat                      ║
║  API:     http://localhost:8000               ║
║  Docs:    http://localhost:8000/docs          ║
╚══════════════════════════════════════════════╝
```

按 `Ctrl+C` 停止，接下来用 systemd 守护进程。

### 3.5 创建 systemd 服务（让后端在后台持续运行）

```bash
sudo vim /etc/systemd/system/deepseek-chat.service
```

写入以下内容：

```ini
[Unit]
Description=DeepSeek Chat Backend Service
After=network.target

[Service]
Type=simple
User=yourname
WorkingDirectory=/home/yourname/projects/deepseek-chat/backend
Environment="PATH=/home/yourname/projects/deepseek-chat/backend/venv/bin"
ExecStart=/home/yourname/projects/deepseek-chat/backend/venv/bin/python main.py
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
# 重新加载 systemd 配置
sudo systemctl daemon-reload

# 启动服务
sudo systemctl start deepseek-chat

# 设置开机自启
sudo systemctl enable deepseek-chat

# 查看服务状态
sudo systemctl status deepseek-chat

# 查看实时日志
sudo journalctl -u deepseek-chat -f
```

### 4.3 调整前端 API 请求地址

部署到生产环境后，前端和 Nginx 在同一台机器上，通过 Nginx 反向代理统一处理，**不需要在前端代码中写死后端地址**。

如果你的前端代码中直接写了 `http://localhost:8000`，需要修改为使用相对路径 `/api`，然后由 Nginx 转发。

Vite 项目通常通过 `vite.config.ts` 中的 proxy 配置处理开发环境代理，生产环境用 Nginx 代理，所以**核心是配置好 Nginx**（见下一章）。

如果你的前端代码中硬编码了后端地址，建议改为：

```typescript
// 使用相对路径，由 Nginx 统一代理
const API_BASE = '/api';

// 而不是：
// const API_BASE = 'http://localhost:8000/api';
```

## 五、Nginx 反向代理配置（核心）

Nginx 是整个部署的枢纽：**接收所有请求 → 静态文件给前端 → /api 请求转发给后端**。

### 5.1 安装 Nginx

```bash
sudo apt install -y nginx

# 启动
sudo systemctl start nginx
sudo systemctl enable nginx

# 验证：浏览器访问 http://服务器IP，看到 Nginx 欢迎页则成功
```

### 5.2 创建站点配置

```bash
sudo vim /etc/nginx/sites-available/deepseek-chat
```

写入以下完整配置：

```nginx
server {
    listen 80;
    server_name 你的服务器IP;   # 或你的域名，如 example.com

    # ========================
    # 前端静态文件
    # ========================
    root /home/yourname/projects/deepseek-chat/frontend/dist;
    index index.html;

    # SPA 路由支持：所有非文件请求都返回 index.html
    location / {
        try_files $uri $uri/ /index.html;
    }

    # ========================
    # 静态资源（JS/CSS/图片等）缓存配置
    # ========================
    location /assets/ {
        # dist 目录下的 assets 文件夹
        alias /home/yourname/projects/deepseek-chat/frontend/dist/assets/;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    # ========================
    # 图片等静态资源（详见第六章）
    # ========================
    location /images/ {
        alias /home/yourname/projects/deepseek-chat/frontend/dist/images/;
        expires 30d;
        add_header Cache-Control "public, max-age=2592000";

        # 可选：开启 gzip（图片本身已压缩，通常不需要）
        # gzip off;
    }

    # ========================
    # 后端 API 反向代理
    # ========================
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;

        # WebSocket / SSE 支持（流式输出必须配置）
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # SSE 流式响应关键配置：关闭缓冲
        proxy_buffering off;
        proxy_cache off;
        proxy_read_timeout 300s;   # 流式超时时间
        proxy_set_header X-Accel-Buffering no;

        # 超时配置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
    }

    # ========================
    # Gzip 压缩
    # ========================
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml text/javascript image/svg+xml;
    gzip_min_length 1000;
    gzip_vary on;
}
```

### 5.3 启用站点

```bash
# 创建软链接到 sites-enabled
sudo ln -s /etc/nginx/sites-available/deepseek-chat /etc/nginx/sites-enabled/

# 删除默认站点（避免冲突）
sudo rm /etc/nginx/sites-enabled/default

# 测试配置是否正确
sudo nginx -t

# 如果显示 "syntax is ok" 和 "test is successful"，则重载
sudo systemctl reload nginx
```

### 5.4 架构流程图

```
浏览器
  │
  ▼
https://你的域名.com  (Nginx :80/:443)
  │
  ├── GET /                   → /frontend/dist/index.html (前端页面)
  ├── GET /assets/*.js        → /frontend/dist/assets/   (JS/CSS)
  ├── GET /images/*.png       → /frontend/dist/images/   (图片)
  │
  └── POST /api/chat          → http://127.0.0.1:8000/api/chat (后端)
      GET  /api/chat/stream   → http://127.0.0.1:8000/api/chat/stream (SSE 流式)
```

## 六、图片等静态资源处理

### 6.1 图片存放位置选择

根据图片数量和用途，有三种方案：

| 方案 | 适用场景 | 存放位置 |
|------|---------|---------|
| **A. 随前端打包** | 少量固定图片（Logo/图标等） | `frontend/dist/images/` |
| **B. 服务器独立目录** | 大量图片、用户上传 | `/var/www/static/images/` |
| **C. 对象存储（推荐）** | 海量图片、需 CDN | 阿里云 OSS / 腾讯云 COS |

### 6.2 方案 A：少量图片随前端打包

```
frontend/
├── public/               # Vite 的 public 目录
│   ├── favicon.svg
│   └── images/           # 把图片放这里
│       ├── logo.png
│       ├── banner.jpg
│       └── ...
```

构建时 `public/` 下的文件会被**原样复制**到 `dist/` 目录。部署后访问路径：

```
http://你的域名/images/logo.png
```

### 6.3 方案 B：大量图片独立存放（推荐）

在服务器上创建独立目录：

```bash
# 创建静态资源目录
sudo mkdir -p /var/www/static/images
sudo mkdir -p /var/www/static/uploads

# 设置权限
sudo chown -R yourname:yourname /var/www/static
sudo chmod -R 755 /var/www/static

# 上传图片到服务器
# 在本地 PowerShell 执行：
scp -r "本机图片文件夹路径/*" yourname@服务器IP:/var/www/static/images/
```

对应的 Nginx 配置：

```nginx
# 在 server 块中添加
location /images/ {
    alias /var/www/static/images/;
    expires 30d;
    add_header Cache-Control "public, max-age=2592000";
}

location /uploads/ {
    alias /var/www/static/uploads/;
    expires 7d;
    add_header Cache-Control "public, max-age=604800";
}
```

### 6.4 方案 C：对象存储（大项目推荐）

如果你的图片非常多（比如用户上传的头像、帖子图片等），建议使用对象存储：

1. **阿里云 OSS**：https://oss.console.aliyun.com/
2. **腾讯云 COS**：https://console.cloud.tencent.com/cos

步骤概要：

1. 开通 OSS/COS，创建 Bucket
2. 设置为公共读
3. 获取访问域名（如 `https://your-bucket.oss-cn-shanghai.aliyuncs.com`）
4. 后端集成 OSS SDK 处理上传
5. 前端直接用 OSS URL 访问图片

### 6.5 批量上传图片到服务器的脚本

创建一个脚本方便批量上传：

```powershell
# upload-images.ps1（Windows PowerShell）
# 将本地 images 文件夹所有图片上传到服务器

$serverIP = "你的服务器IP"
$user = "yourname"
$localPath = ".\images\*"
$remotePath = "/var/www/static/images/"

scp -r $localPath ${user}@${serverIP}:${remotePath}
Write-Host "图片上传完成！"
```

## 七、域名 + HTTPS 配置（SSL 证书）

### 7.1 域名解析（DNS）

1. 登录你的域名服务商（阿里云 DNS / 腾讯云 DNS）
2. 添加 A 记录：
   - **主机记录**：`@`（代表根域名）或 `www`
   - **记录类型**：`A`
   - **记录值**：你的服务器公网 IP
   - **TTL**：600（默认）

示例：

| 主机记录 | 记录类型 | 记录值 | TTL |
|---------|---------|--------|-----|
| `@` | A | `123.456.789.10` | 600 |
| `www` | A | `123.456.789.10` | 600 |

修改后等待 DNS 生效（通常 1-10 分钟）。

### 7.2 安装 Certbot（Let's Encrypt 免费 SSL）

```bash
# 安装 certbot
sudo apt install -y certbot python3-certbot-nginx

# 先确保 Nginx 配置中的 server_name 已改为你的域名
sudo vim /etc/nginx/sites-available/deepseek-chat
# 修改：server_name example.com www.example.com;

# 测试配置
sudo nginx -t
sudo systemctl reload nginx

# 自动获取并配置 SSL 证书
sudo certbot --nginx -d example.com -d www.example.com
```

按提示操作：
- 输入邮箱（用于证书到期提醒）
- 同意服务条款
- 选择是否重定向 HTTP 到 HTTPS（建议选 2，自动重定向）

### 7.3 证书自动续期

```bash
# certbot 会自动添加定时任务，验证一下
sudo systemctl status certbot.timer

# 手动测试续期
sudo certbot renew --dry-run
```

## 九、防火墙与安全组配置

### 9.1 云服务器安全组（在云服务商控制台配置）

登录你的云服务器控制台（阿里云 ECS / 腾讯云轻量服务器），找到**安全组**配置，确保以下端口开放：

| 端口 | 协议 | 用途 | 来源 |
|------|------|------|------|
| 22 | TCP | SSH 远程登录 | 0.0.0.0/0 |
| 80 | TCP | HTTP | 0.0.0.0/0 |
| 443 | TCP | HTTPS | 0.0.0.0/0 |
| 8000 | TCP | 后端 API（**仅测试时开，上线后关掉**）| 0.0.0.0/0（测试完删除）|

> ⚠️ **安全提示**：上线后务必关闭 8000 端口的公网访问！后端只需 127.0.0.1 本地访问即可，通过 Nginx 代理对外服务。

### 9.2 服务器内部防火墙（Ubuntu ufw）

```bash
# 开启 ufw
sudo ufw enable

# 允许 SSH
sudo ufw allow 22/tcp

# 允许 HTTP 和 HTTPS
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp

# 查看规则
sudo ufw status numbered
```

## 十一、常见问题排查

### Q1：访问页面显示 502 Bad Gateway

**原因**：Nginx 无法连接后端服务。

```bash
# 检查后端是否在运行
sudo systemctl status deepseek-chat

# 检查端口是否监听
sudo netstat -tlnp | grep 8000

# 查看 Nginx 错误日志
sudo tail -50 /var/log/nginx/error.log
```

### Q2：刷新页面后显示 404

**原因**：SPA 路由配置问题。

确保 Nginx 配置中有这一行：
```nginx
location / {
    try_files $uri $uri/ /index.html;
}
```

### Q3：API 请求返回 CORS 错误

**原因**：后端 `ALLOWED_ORIGINS` 中没有配置当前访问的域名。

修改 `config.py`：
```python
ALLOWED_ORIGINS = [
    "http://你的域名",
    "https://你的域名",
    "http://服务器IP",
]
```

然后重启后端：
```bash
sudo systemctl restart deepseek-chat
```

### Q4：流式输出不工作（一次性返回）

**原因**：Nginx 缓冲未关闭。

确保 API 代理配置中有：
```nginx
proxy_buffering off;
proxy_cache off;
```

### Q5：图片加载不出来

```bash

ls -la /var/www/static/images/

# 检查 Nginx 配置中的路径是否正确
sudo nginx -t

# 查看 Nginx 访问日志
sudo tail -20 /var/log/nginx/access.log
```
