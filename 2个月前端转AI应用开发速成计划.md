# 前端开发转 AI 应用开发 — 2 个月速成学习计划

> **背景与定位**：基于你具备 6 年前端开发经验及扎实的工程化能力，现有的 6 个月计划较为平缓。如果你希望在 **2 个月（8 周）** 内快速形成战斗力并完成转型，我们需要**大幅压缩非核心基础学习时间**，发挥前端优势（如直接使用 TypeScript/Vercel AI SDK 构建应用），并重点攻克 AI 特有的概念（RAG、Agent、Prompt 工程）。

---

## 🧐 现有内容评估与补充建议

你原有的计划非常系统，但针对“短期速成”和“当前最新技术栈”，我建议补充以下核心内容：

### 1. 补充：本地大模型与低成本开发
原计划侧重于调用 OpenAI 等商业 API。在实际开发和测试中，强烈建议加入 **Ollama** 或 **LM Studio**，在本地运行开源大模型（如 Llama-3、DeepSeek-Coder），实现零成本开发测试。

### 2. 补充：AI 编排平台（BaaS/Low-code）
除了纯手写 LangChain 代码，目前业界极为流行使用 **Dify**、**Coze** 或 **FastGPT** 等开源/商业化 AI 工作流编排平台。前端开发者可以通过这些平台快速配好后端 Agent 和 RAG，然后专注于开发高度定制化的前端 UI。

### 3. 补充：前端专属 AI 工具链
原计划 Python 比重较大。对于前端而言，**Vercel AI SDK**（配合 Next.js）是目前全栈 AI 应用事实上的标准。你应该将大量精力放在如何用前端技术栈无缝对接大模型流式输出（SSE）。

### 4. 补充：国产优质大模型生态
重点加入 **DeepSeek（深度求索）** 或 **通义千问** 等极具性价比的国产模型 API 对接，这在国内 AI 应用开发中非常关键。

## 🚀 2 个月（8周）高强度速成计划

**总体策略**：每周完成一个可运行的 Demo，以战代练。利用前端现成脚手架（如 Next.js），不要在基础语法上死磕。

### 阶段一：破冰与大模型对话（第 1-2 周）
**目标**：掌握大模型 API 调用、流式输出机制，跑通第一个 AI Chat 页面。

* **Week 1：Python 速成与 LLM API 基础**
  * **Python 极简上手**：基础语法、虚拟环境 (`venv`)、依赖包管理 (`pip`)，重点学习 `FastAPI` 写简单的接口。
  * **LLM 核心概念**：Context Window、Token 计费、Temperature。
  * **API 实战**：对接 DeepSeek 或 OpenAI API，掌握多轮对话结构（System/User/Assistant Messages）。
  * **产出**：一个纯命令行的多轮对话终端程序（Python）。
  * **补充工具**：安装 Ollama 并本地运行 `llama3` 或 `qwen`。

* **Week 2：前端接入与 Prompt 工程**
  * **Vercel AI SDK**：在 Next.js 项目中引入 `ai` 和 `@ai-sdk/react`，实现打字机效果的流式输出（Streaming）。
  * **Prompt 进阶**：学习 CoT（思维链）、Few-shot、让大模型强制输出 JSON（JSON Mode）。
  * **产出**：一个带 UI 的网页版 AI 聊天应用，支持历史记录上下文（可无数据库，暂存 LocalStorage）。

### 阶段二：给 AI 挂载知识库（第 3-4 周）
**目标**：理解向量概念，能够手撸一个 RAG（检索增强生成）系统。

* **Week 3：向量与检索基础**
  * **Embedding**：理解文本如何转成高维向量，以及余弦相似度。
  * **向量数据库**：使用轻量级的本地库 ChromaDB，或者云端的 Pinecone / Supabase (pgvector)。
  * **数据处理**：文档加载（PDF/Markdown）、文本切片（Text Splitting）。
  * **产出**：通过 Python 写一个脚本，读取本地 Markdown 并在其中进行语义搜索。

* **Week 4：RAG 应用组装**
  * **RAG 流程串联**：用户提问 -> 向量检索 -> 组装 Prompt -> 大模型生成。
  * **LangChain 入门**：使用 LangChain（Python 或 JS 版本均可）来简化上述链条。
  * **产出**：**“AI 文档助手”** Web 页面。用户上传文本，可以针对文本内容提问并给出精准回答。

### 阶段三：让 AI 拥有执行力（第 5-6 周）
**目标**：掌握 Function Calling 和 Agent 的核心思想。

* **Week 5：Function Calling (工具调用)**
  * **原理**：理解 LLM 如何决定调用哪个工具，以及如何解析其输出参数。
  * **实战**：定义一个“查询天气”和“获取网页内容”的函数，提供给大模型调用。
  * **产出**：一个具备联网搜索能力的聊天机器人。

* **Week 6：Agent 与工作流编排**
  * **Agent 原理**：ReAct 模式（Reason + Act），大模型自主规划任务步骤。
  * **LangGraph 基础**：用状态机（State Graph）的思维构建稳定可控的多步 Agent 流程。
  * **效率工具体验**：部署或注册一个 **Dify** 账号，用可视化界面快速拖拽一个 Agent，对比与写代码的差异。
  * **产出**：一个可以自动拆解任务、调研并生成报告的 **“AI 调研员”** 后端接口。

### 阶段四：全栈项目落地与商业化思维（第 7-8 周）
**目标**：开发一个完整的 AI SaaS 级应用，作为面试作品集。

* **Week 7：全栈 AI 项目开发**
  * 结合前 6 周所学，做一个完整的项目（例如：**智能简历优化工具** 或 **AI 小说推文生成器**）。
  * **架构**：Next.js (前端 + BFF API) + FastAPI (复杂 Agent 逻辑) + PostgreSQL (用户数据)。
  * 重点解决用户鉴权（Auth）、状态管理和良好的交互反馈（Loading状态、骨架屏）。

* **Week 8：部署、监控与复盘**
  * **云端部署**：前端部署至 Vercel，后端 Python 服务部署至 Render 或 Railway。
  * **可观测性**：接入 LangSmith 观察每一次 LLM 调用的 Prompt、耗时和 Token 消耗。
  * **面试准备**：整理这 8 周的 3 个核心产出（对话基础、RAG 问答、Agent 工具），输出为技术博客，更新简历。

## ⏱️ 时间管理建议 (每周 15-20 小时)

要在 2 个月内完成，你的时间颗粒度需要更细：

| 日期 | 学习时间 | 内容安排 |
| :--- | :--- | :--- |
| **周一至周四** | 每天 1.5 ~ 2 小时 | 晚上集中精力看核心原理，**只手敲最关键的 Demo 代码**，不要在 UI 细节上浪费时间。 |
| **周五** | 休息或复盘 | 整理本周踩过的坑，写一篇简短的博客或学习笔记。 |
| **周末** | 每天 4 ~ 6 小时 | 大块时间用来做“产出项目”，调试接口、合并前后端代码。 |

> **关键提醒**：作为一个资深前端，你在做这个计划时的最大陷阱是**过度沉迷于前端 UI 交互的完美**。请一定克制！这 2 个月的核心是**死磕 AI 逻辑（RAG 质量、Prompt 调优、Agent 稳定性）**。UI 只要用 Tailwind 或 Shadcn/ui 套用现成组件，看得过去就行。把时间留给最核心的 AI 价值！
