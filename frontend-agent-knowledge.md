 ## 一、LLM — 大模型到底是个啥？

 **LLM 的核心就一句话：预测下一个词。**

 它并不"聪明"，也不理解人话，它的工作方式是：被喂了整个互联网的数据，然后疯狂做"补全"。

 举个例子，你输入"猴子喜欢吃"，它会在海量训练数据里算出来，候选列表里"香蕉"的概率最大，于是返回"香蕉"。写诗、写代码、画图都一样——都是根据 prompt 做补全，只不过结构更复杂。

 几个关键概念：

 - **Chat 模式 vs Completion 模式**：早期 GPT-3 是纯文本补全（Completion），现在 GPT-3.5/4 都是对话模式（Chat），输入消息列表，输出新内容
 - **MoE（混合专家模型）**：模型太大了每次跑全部太贵，就拆成多个子模型，每次只激活其中几个，成本更低
 - **微调（Fine-tuning）**：不是重新训练整个模型，是调整其中很小一部分参数，改变它的"预测倾向"

 说白了，LLM 的"聪明程度"取决于参数数量——参数就像神经元的数量。参数越多，训练和运行成本都越高，但预测得也更准。

 ## 二、Prompt Engineering — 提示词工程，其实就是"怎么把话说明白"

 **AI 生成的质量，严重依赖你给它的提示词。** prompt 模糊，答案一定模糊。

 作者说得很实在：严格来讲 Prompt Engineering 不是什么高深技能，**它就是一些沟通方式**，很容易理解。

 实际开发中常见的做法：

 - **约束范围**：比如 GitHub Copilot 只专注编程领域，你问别的它不回答——这就是 prompt 约束的效果
 - **CoT（思维链）**：引导模型按你的思路一步步思考，而不是直接跳结论
 - **格式化输出**：规定 AI 的输出是 JSON、表格还是 Markdown
 - **包装和转述**：用户输入什么，你不会原封不动传给 AI，而是要做很多包装和改写

 **关键认知：** 每次调 AI 接口之前，你都要认真想 prompt 怎么写。甚至可以用 AI 来写 prompt。

 ## 三、LangChain.js — 前端/Node 做 AI 的首选框架

 前端用 Node.js 开发 AI 应用，LangChain.js 是首选。它的生态很完整：

 | 组件 | 干什么用的 |
 |---|---|
 | LangChain | 核心框架，封装了 LLM 调用、Prompt 管理、工具集成 |
 | LangGraph | 自定义 Agent 工作流，让 Agent 按你的逻辑跑 |
 | LangSmith | 跟踪和分析 Agent 每一步的行为，调试神器 |

 作者还写了几篇实战文章（可以做学习参考）：
 - 30 行代码用 LangChain.js 开发第一个 Agent
 - 用 LangChain.js 实现 RAG 知识库语义搜索
 - 用 LangChain.js 实现掘金"智能总结"

 ## 四、RAG — 让 AI 学会"查资料"再回答

 RAG（Retrieval-Augmented Generation）的中文叫**检索增强生成**。核心思路很简单：

 > 不要让 AI 凭记忆瞎编，先让它去数据库里查资料，再根据查到的东西回答。

 三步走：

 ```mermaid
 flowchart LR
     A[把资料拆成向量存起来] --> B[用户提问时去查最相关的资料]
     B --> C[把查到的资料发给 AI，让它基于这些内容回答]
 ```

 ### 最难理解的概念：向量（Vector）

 向量就是**坐标**。你熟悉的是二维（x,y）或三维坐标，而一段文本可以被转换成几百维的坐标（float 数组）。

 **两个向量的距离 = 两段文本的相似度。**

 这比传统搜索引擎强在哪？Elastic Search 做的是**关键词匹配**，"教程"匹配不到"课程"。但向量做的是**语义匹配**，"教程"和"课程"虽然字不同，但意思相近，向量距离就小。

 ### 向量数据库怎么选？

 | 阶段 | 推荐 |
 |---|---|
 | 开发阶段 | Chroma（轻量，够用） |
 | 生产部署 | Pinecone 或 Supabase（都有免费额度） |

 ## 五、Agent — 前面所有东西的组合体

 Agent 是一个**综合体**，它把前面讲的东西都串起来了：

 ```mermaid
 flowchart TB
     LLM[大模型 — 负责思考和生成] --> Agent
     WF[Workflow 工作流 — 定义节点和判断逻辑] --> Agent
     Tools[工具 — 调用外部服务] --> Agent
     Mem[Memory 记忆 — 记录会话和关键信息] --> Agent
 ```

 简单说，Agent = 大脑（LLM）+ 流程（Workflow）+ 双手（Tools）+ 记事本（Memory）。

 像 Dify、Coze、Flowise 这些平台就是可视化搭建 Agent 工作流的工具，拖拖拽拽就能配出一个 Agent。

 ## 六、MCP — 统一工具调用的"普通话"

 MCP（Model Context Protocol）是由 Anthropic 提出的**模型上下文协议**。它的作用很简单：

 > 给 AI 调用外部工具定一个统一标准，就像 USB 接口一样——不管插什么设备，接口都一样。

 以前每个模型调工具的方式都不一样，有了 MCP 就统一了。你要掌握的是：

 - 能用现成的 MCP Server（比如编程用的各种 MCP 工具）
 - 也能自己开发 MCP Server 和 Client

 ---

 ## 七、多模态 — AI 不只是聊天的了

 现在的 AI 应用可不只是打字聊天：

 - **输入**：可以上传图片、PDF、Word、音频、视频
 - **输出**：AI 也能生成图片、PDF、音频、视频

 非文字的输出通常通过 **Artifact** 形式展示。比如你用 Claude 让它生成一个 HTML 网页，它会在右侧直接展示渲染效果，还能直接发布上线。

 **不同模型擅长的模态不一样，API 的调用方式和参数写法也不同。**

 ---

 ## 八、还有哪些值得关注的前沿方向？

 文章最后提到几个正在发展的方向：

 - **Multi-agent（多智能体架构）**：多个 Agent 协作完成复杂任务
 - **A2A 协议**：Agent 和 Agent 之间的通讯协议
 - **Context Engineering（上下文工程）**：怎么更好地管理和使用上下文
 - **AG-UI 协议**：Agent 和前端 UI 之间的通讯协议

 这些是前沿，暂时不需要都学，但要知道有这些东西。

 ---

 ## 三句总结

 1. **LLM 本质就是"预测下一个词"**，不需要自己训模型，核心是把 prompt 写明白、把工程搭好
 2. **前端开发 Agent 的路线很清晰**：LLM API 调用 → Prompt 工程 → LangChain.js → RAG → Agent 组合 → MCP → 多模态，每一步都可以用 Node.js 搞定
 3. **向量是理解 RAG 的关键门槛**，把它理解成"坐标 + 距离 = 语义相似度"，后面的都好办了
